# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability within this project, please send an email to logeshkannan19@example.com. All security vulnerabilities will be promptly addressed.

## Security Best Practices

- Never commit sensitive data (API keys, passwords, credentials) to the repository
- Use environment variables for sensitive configuration
- Validate all user inputs in the dashboard
- Keep dependencies up to date

## Data Security

- The project uses synthetic data for demonstration purposes
- In production, ensure proper database encryption and access controls
- Follow GDPR/CCPA compliance when handling real customer data
